1. source源文件 决定博客的文章内容

    2. CNAME用于DNS域名转发
2. scaffolds脚手架（即模版复用的内容）
    3. draft草稿
    4. page页面
    5. post文章
3. themes主题
    4. 布局 layout
        5. _portial 组件
        6. _widget 定制化内容
        7. 各个模块
    5. 内容 source
        6. 资源文件 media
        7. css、js
    6. 配置 config
4. public渲染后的项目
   5. 资源+页面信息

[//]: # (博客如何运行？)
1. 在source目录下写md源文件
2. 生成百度、谷歌访问的url
3. 更新search、主页等文件内容
4. 运行本地服务器，访问文章：模版决定页面固定展示的文案+post.js渲染页面动作，并从资源文件读取图片等信息+渲染时引用其他组件，则执行其他组件内容，如果有定制内容，同样读取执行
5. 为了方便操作，可以加sh对重复性的命令做集成，也可以做定制化的特效
